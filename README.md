# Jesus Mendoza — VA Portfolio

This is my virtual assistant portfolio site. It's one self-contained file (`index.html`) — no build tools, no other files needed.

## How to put this live on GitHub Pages (free hosting)

1. Go to [github.com](https://github.com) and log in (or make a free account if you don't have one).
2. Click the **+** icon top right → **New repository**.
3. Name it something like `portfolio` or `jesus-mendoza-va`. Keep it set to **Public**. Leave everything else unchecked (don't add a README — we already have one).
4. Click **Create repository**.
5. On the new repo page, click the link that says **uploading an existing file**.
6. Drag both `index.html` and `README.md` into the upload box.
7. Scroll down and click **Commit changes**.
8. Click the **Settings** tab (top of the repo) → **Pages** (left sidebar).
9. Under **Build and deployment** → **Source**, pick **Deploy from a branch**.
10. Under **Branch**, pick `main` and folder `/ (root)`, then click **Save**.
11. Wait a minute or two, then refresh that Settings → Pages screen. GitHub will show your live link, something like:
    `https://yourusername.github.io/portfolio/`
12. That link is your live portfolio — put it in your applications, LinkedIn, and email signature.

### Important
The file has to stay named exactly `index.html`. That's the name GitHub Pages looks for as the homepage — if you rename it, the site won't load right.

### Updating it later
Whenever you want to change something, go back into the repo, click on `index.html`, click the pencil icon (edit), make your change, and commit again. The live site updates automatically within a minute or two.
